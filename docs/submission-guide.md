# Submission Guide

## TL;DR

1. Build in your own GitHub repo (public, or private with judge access).
2. Before the deadline, open an Issue in the [`submissions`](https://github.com/abu-dhabi-ai-proptech-challenge/submissions) repo using the **Project Submission** form.
3. One submission per team. Late submissions are not judged.

## Step by step

### 1. Get your project into a repo

- Use your own repo, or generate one from the [`project-template`](https://github.com/abu-dhabi-ai-proptech-challenge/project-template) ("Use this template" button).
- Make it **public** if you can. If it must stay private, add `@evoost-judge` as a collaborator so judges can review it.
- Make sure the README explains how to run it.

### 2. Prepare your links

You'll need:

| Item | Required |
|---|---|
| GitHub repo link | ✅ |
| Demo link (deployed app, video, or notebook) | ✅ — at least one |
| Slides link | Optional but recommended |

For the demo link: a deployed URL (Vercel, Hugging Face Spaces, etc.) is best; a 2–3 minute screen recording is a solid fallback if deployment is fighting you.

### 3. Open the submission Issue

Go to **[`submissions` → Issues → New Issue → Project Submission](https://github.com/abu-dhabi-ai-proptech-challenge/submissions/issues/new/choose)** and fill in the form:

- Team name and members
- Track (one of the four)
- Project summary, problem, and solution
- Tech stack
- Repo, demo, and slides links
- **What you built during the hackathon** (be specific — judges read this)
- Setup instructions

Submit the Issue. That's it — the timestamp on the Issue is your submission time.

### 4. After submitting

- Organizers label and acknowledge your submission.
- You can edit the Issue to fix typos or broken links **before the deadline**. Substantive changes after the deadline are not considered.
- Watch the Issue for any questions from judges.

## Checklist before you hit submit

- [ ] Repo link works in an incognito window (or judge access granted)
- [ ] README says how to run the project
- [ ] Demo link works
- [ ] The "what we built today" section is honest and specific
- [ ] Track selected matches what you built

## Common mistakes

- Submitting a repo that's empty except for a README
- Demo links that require a login nobody has
- Describing the roadmap instead of what exists
- Submitting at deadline-minus-30-seconds — the form takes a few minutes; start early

Questions? Open a **Question** issue in `submissions` or ask on Discord (https://discord.gg/jy3QDxQ3jK).
